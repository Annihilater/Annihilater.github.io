不能直接照官方文档上面的来，不然安装的时候可能会出现没有访问权限的问题。

首先先看一下用户id，以我的是root为例，可以使用 id root 命令查看 root 用户的 uid、gid

  • BucketName：Bucket 的名字
  • ~/ossfs/klause-seafile ：OSS 要挂载的目录
  • uid：指定用户的 uid
  • gid：指定用户的 gid
  • -ourl= ：是阿里云 OSS 所在地区的外网Endpoint，可以在阿里云帮助文档访问域名和数据中心查看
  • allow_other：表示允许其他用户访问
  • umask：设置挂载文件夹的权限，默认设置 777，会被提示不安全，挂载为770权限，不知道为啥这里 0 代表 7，7 表示0

某次 CentOS 中挂载遇到的一些问题

如果 sudo ossfs 找不到 ossfs 命令，可以使用 where ossfs 查看 ossfs 命令的路径，比如路径为 /usr/local/bin/ossfs ，然后使用 sudo /usr/local/bin/ossfs 执行命令。

可以使用 -o passwd_file= 指定秘钥文件，而不必须使用 /etc/passwd-ossfs 作为秘钥文件，这样可以避免 /etc/passwd-ossfs 文件因权限无法访问的问题。
