Pyecharts 超火动态排序图：新冠确诊数动态排序图

之前使用 Matplotlib 实现了动态排序图，本文使用 pyecharts 库来实现一个版本。pyecharts 有一个缺陷是无法为不同的国家配色，所有国家都是同一个颜色。

思路

  1. 从数据源提取有用数据
  2. 从有用数据中提取出一天的数据，按照国家分组求和
  3. 根据每天各国确诊病例数画出柱形图
  4. 使用 pyecharts.charts.Timeline 将每天的柱形图绘制成动画

绘制动画

因为 pyecharts 绘制动画的原理是将一张张图表添加到 Timeline 中，再一起渲染出来。

原理是将每天的确诊数前十的国家数据提取出来，将国家列表分给 x 轴，将确诊人数分给 y轴，使用柱形图画出来。

再将每天的柱形图添加到 Timeline，最后一起在 jupyter 内渲染。

pyecharts 是基于 echarts 开发的，echarts 可以为柱形图的每个柱子配置不同搞得颜色，pyecharts 也支持的，但是在动态排序图中需要柱子的颜色随着国家确诊人数的变化实现在排序中变动是不可能的，因为 echarts 实现不同柱子配不同颜色的原理是使用 JavaScript，可以在 JavaScript 函数内定制每根柱子的颜色，但是颜色要基于国家就需要在传确诊数的同时将国家列表传给 y 轴，echarts 没有提供这种接口，后来查看渲染生成的 HTML 页面发现，可以修改 HTML 页面的 JS 脚本来实现自动配颜色，但操作失败，因为无法实现确诊数与国家之间的关联。

HTML 页面内的国家与确诊数是同一个 JavaScript 对象的不同子对象，暂时无法访问到，可能是对 JavaScript 理解不够深吧。
