问题 2

有一个国家发现了 5 座金矿，每座金矿的黄金储备量不同，需要挖坑的工人数也不同。参与挖矿工人的总数是 10 人。每座金矿要么全挖，要么不挖，不能派出一半人挖取一半金矿。要求用程序求解出，要想得到尽可能多的黄金，应该选择挖取哪几座金矿？

金矿名 矿产/金 用工数/人
金矿 1 400 5
金矿 2 500 5
金矿 3 200 3
金矿 4 300 4
金矿 5 350 3
工人数 0人 1人 2人 3人 4人 5人 6人 7人 8人 9人 10人
金矿 1 0 0 0 0 0 400 400 400 400 400 400
金矿 2 0 0 0 0 0 500 500 500 500 500 900
金矿 3 0 0 0 200 200 500 500 500 700 700 900
金矿 4 0 0 0 200 300 500 500 500 700 800 900
金矿 5 0 0 0 350 350 500 550 650 850 850 900

也是动态规划的思路，有 5 座金矿，假如前面 4 座先不管，就看第 5 座，那就有两种情况：

  • 不挖：前 4 座分配 10 个人挖，获取最大黄金量
  • 挖：前面 4 座分配 10-3 个人挖，获取的黄金量 = 挖前 4 座获取的黄金量 + 350 金

显然上图显示了本问题的最优子结构。接下来就是分析最优子结构与最终问题的关系：4 座金矿的最优选择与 5 座金矿的最优选择之间的关系？

显然 5 座金矿获取的黄金量有两种算法：

  • 4 座金矿 10 个人的最优选择
  • 4 座金矿 7 个人的最优选择 + 第 5 座金矿的含金量

所以 5 座金矿的最优解应该是两者的最大值。

建模

金矿数量设为 n，工人数设为 w，金矿的含金量（按照金矿顺序）设为数组 g，金矿的用工量（按照金矿数量）设为数组 p，最优选择设为函数 F（n，w）

显然可以得出

确定边界

只有第一座金矿的时候，没得选：

  • w > p[0]用工数（第一座金矿的用工数）：挖， F(n, w) = g[0]
  • w < p[0]（第一座金矿的用工数）：不挖， F(n, w) = 0

状态转移方程

  • F(n,w) = 0 (n<=1, w<p[0])
  • F(n,w) = g[0] (n==1, w>=p[0])
  • F(n,w) = F(n-1,w) (n>1, w<p[n-1])
  • F(n,w) = max(F(n-1,w), F(n-1,w-p[n-1])+g[n-1]) (n>1, w>=p[n-1])

解决代码

解决方法代码中的 pre_results、results 保存的就是行结果。

动态规划的方法利用两层迭代，逐步推导最终结果。在外层的每一次迭代，表示按照金矿顺序增加一座金矿，也就是对表格每一行的迭代过程，都会保留上一行的结果数组 pre_results，并循环计算当前行的结果数组 results。

时间复杂度 $O(n*w)$

空间复杂度 $O(w)$

需要注意的是，当金矿只有 5 座的时候，动态规划的性能优势还没有体现出来。当有 10 座金矿，甚至更多金矿时，动态规划就明显具备了优势。

拓展思考

如果把题目改一下，总工人数变成 1000 人，每个金矿的用工数也相应增加，这时候该如何实现最优选择呢？

  • 动态规划方法
    • 时间复杂度 $O(5*1000)$
    • 空间复杂度 $O(1000)$
  • 简单递归方法
    • 时间复杂度 $O(2^5)$ （5 座金矿）

显然 1000 个工人的时候，动态规划的性能远不如简单递归。

  • 动态规划方法的时间和空间都和 w 成正比
  • 简单递归方法的时间却和 w 无关

所以当工人数量很多的时候，动态规划反而不如递归。所以说，每一种算法都没有绝对的好与坏，关键看应用场景。
