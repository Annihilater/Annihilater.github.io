本章介绍最常见的sql注入攻击、 xss攻击和csrf攻击的原理以及防护

12-1 sql注入攻击与防范

sql 注入的危害：

  1. 非法读取、篡改、删除数据库中的数据
  2. 盗取用户各类敏感信息，获取利益
  3. 通过修改数据库来修改网页上的内容
  4. 注入木马等

12-2 xss攻击原理及防范

xss 跨站脚本攻击（Cross Site Scripting）的危害：

  1. 盗取各类用户账号，如网银账号、各类管理员账号
  2. 盗窃企业重要的具有商业价值的资料
  3. 非法转账
  4. 控制受害者机器向其他网站发起攻击、注入木马等等

xss 攻击原理

xss 攻击的流程：如果服务器没有对 name 参数做校验

黑客将 url 中添加 js 代码：

  1. 黑客给用户的发送方式有很多种，比如：QQ 消息、邮件等等。

  2. 受害者看到链接就会点击链接，因为它信任这个链接

  3. 然后服务器就会执行 js 脚本，还会将执行结果原模原样返回受害者浏览器

    如果黑客在 js 脚本中加入了其他逻辑将截取到的用户信息发送给黑客，黑客就能拿到用户的 cookie、session 等等重要信息

xss 攻击防护

  • 首先对用户输入的地方和变量做仔细检查，比如长度等，对以下字符等做过滤

  • 避免直接在 cookie 中泄露用户隐私，例如：用户名、email、密码等

  • 可以将系统 ip 与 cookie 绑定，来降低 cookie 泄露后的危险（cookie 泄露后由于服务器检测到当前 cookie 的 ip 发生变化，立即使当前 cookie 失效）

  • 尽量才有 POST 而非 GET 的表单提交方式

12-3 csrf攻击与防范

csrf 跨站请求伪造（Cross-site request forgery）的危害：

  1. 以你的名义发送邮件
  2. 盗取你的账号
  3. 购买商品
  4. 虚拟货币转账

csrf 攻击原理

用户在第5步的时候会自动带上 A 站点的 sessionid 或者 cookie 等中啊哟信息。（当然这得到这些信息有效的情况下才能攻击成功，如果服务器设置关闭标签页这些信息就失效的话，那么一但用户关闭 A 站的标签页再去访问 B 站就不会攻击成功）

即使A 站服务器后台限制使用 POST 方法来访问，黑客可以自定义一个 form，在用户加载完页面之后自动执行一段 POST 代码，同样攻击有效。

csrf 防护

Django 要求每一个 form 表单都需要添加 {% csrf_token %}，每次表单提交的时候都需要将 csrf_token带回服务器 Django 会自动做校验。黑客网站是没有这个 csrf_token的，即使黑客站点自己构建了 csrf_token无法通过 Django 服务器的校验。
