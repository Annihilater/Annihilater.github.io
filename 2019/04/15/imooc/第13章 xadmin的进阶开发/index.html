设置字段搜索

添加课程的时候，需要下拉选择课程机构，当机构数目非常多的时候就非常麻烦，比如1000个难道每次1000个拉到底吗？显然不方便。那么可不可是一种搜索的的样式展现呢？比如在框中输入 上自动显示机构名称包含 上的的机构。答案是肯定的，只需要在对应的 Admin设置一个字段就可以了。

这里的课程机构是课程 model 的外键，应该是在外键链接的 modelAdmin 里面设置。

这里是课程机构的下拉，所以在 imooc/apps/organization/adminx.py 内设置： relfield_style = ['fk-ajax']就可以了。

fk 的意思就是 ForeignKey，一个 model A，当其他 model B 有外键指向 model A 的时候，它的展现方式是以一种 ajax加载的方式完成的，也就是我们所要的搜索效果。

同一个 model 注册两个管理器

在首页一个轮播课程，判断课程是否为轮播课程的方法就是课程下有个 is_banner的是 True或 False，在 xadmin 管理后台这个值的设置入口比较深，是在课程编辑页面，如果我们需要对轮播课程和非轮播课程做统一管理就比较麻烦，可以将轮播课程和非轮播课程区分开吗？实际上在数据库里轮播课程和非轮播课程是在一张表里的，这里我们就需要使用两个管理器来管理一张表。

显然我们不能用一个 model 去注册两个管理器，因为注册第二个管理器的时候会报该 model 已经注册的错误，所以我们需要再定义一个 model，比如 BannerCourse，BannerCourse 继承 Course，字段基本不需要改动，需要改的地方就是 Meta 信息，表的名字需要改下，Meta 内有一个 proxy 字段需要设置为 True，这个意思表示该 model 不在数据库内创建数据表，否则 Django 会在数据库再生成一张表。

好了现在可以为 BannerCourse 注册管理器了，BannerCourse 管理器的注册跟 Course 是一样的，字段信息全部拷贝过来就可以了。

注册好之后打开 xadmin 管理后台一看确实出现了 BannerCourse 管理器页面，但是问题来了，BannerCourse 管理器和 Course 管理器的内容完全一样，那么如何在两个管理器展示不同的内容呢？

显示自定义的 html 代码

上面既然能将对象方法的调用结果显示出来，那么如果在对象的方法内写入 html 代码，比如跳转链接，肯定一样会在管理器页面显示跳转链接。

这里必须要调用 mark_safe方法，否则 Django 会对字符串中的特殊符号，比如 >做文本转换，显示到浏览器就不是我们设定的样式了，使用 mark_safe是告诉 Django 这段文本是安全的，不需要做转换，这样的话文本才能完整的显示在前端。

坑 1：DjangoUeditor 对 python3 支持不好

理论上应该已经可以使用 DjangoUeditor编辑器了，但是遇到点小插曲，由于 DjangoUeditor是基于 python2.7 的对 python3 的支持有些问题，会导致

所以搜了下网上的解决办法有两种：

  1. 修改 widget.py文件
  2. 使用网上别人修改好的 DjangoUeditor版本：https://github.com/twz915/DjangoUeditor3

采用的是第二种方法：

  1. 先卸载之前安装的DjangoUeditor

  2. 从 Github 下载别人修改好的DjangoUeditor的源码，解压从源码安装DjangoUeditor，cd 到解压后的文件夹目录，开启虚拟环境：

安装成功就可以重启服务器试试看了，发现已经可以使用DjangoUeditor富文本编辑器了。

坑 2：显示效果不对

但是还有个问题，就是前端显示的效果不对。实际上这是 Django 对富文本自带的的防御机制，Django 会对某些字符做转义，比如 >，所以我们需要在前端显示的地方关闭告诉 Django 这里的显示不需要转义子字符。
