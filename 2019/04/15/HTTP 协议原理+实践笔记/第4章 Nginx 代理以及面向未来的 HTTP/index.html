    监听的域名：yuming.com
    代理服务器指向的地址：127.0.0.1:8888

  2. 修改代理的 HTTP 头

    这个设置是为了显示浏览器原始 HTTP 头，如果不设置，真实服务器接收到的所有 HTTP 头都会是我们在 proxy_pass 里设置的头。

$host 是 nginx 里的变量

如果有一些 HTTP 头通过代理之后，你发现写入服务的并不是原始的 HTTP 头，有可能是被中间代理修改了。

中间代理是可以做任何它想要做的事情，至少在 HTTP 协议层面是这样的。如果使用 HTTPS 的话，那么整个传输过程是加密的，代理是无法修改传输内容的。

大家可能有体会，当我们在手机上浏览网页的时候可能会弹出中国移动、中国联通的广告，这些广告就是因为中国移动、中国联通是 HTTP 的代理，服务端返回内容的时候被它们解析了，它们在内容里插入了一段代码去展示它们的广告。

4-2 Nginx 代理配置和代理缓存的用处

配置 nginx 缓存

代理缓存的作用：

代理缓存是在代理服务器中设置的，每一个新的请求都会经过代理，如果代理服务器中已经缓存过一次了，后续的用户请求都可以直接使用代理缓存。

它更高效的为一些通用的缓存内容提供了缓存功能。这就是代理缓存最大的意义。

参数解释：

  • max-age：缓存时间
  • s-maxage：代理缓存专用的缓存时间
    如果 max-age 和 s-maxage 同时使用时，代理服务器会选择使用 s-maxage，浏览器会选择使用max-age
  • private：限制代理服务器不能使用缓存，只要出现 private，代理服务器就不能执行缓存操作
  • no-store：所有地方都不能执行缓存操作
  • Vary：用来指定发送请求的时候，只有 Vary 指定的这些 HTTP 头，它们的值是相同的情况下，才会允许使用缓存。
    这就是验证头信息，如果指定的头信息不一样，那么它就不能使用缓存。应用场景：
    比如我们使用了服务器缓存，但是我们需要返回的数据需要根据 User-Agent 来返回，因为移动端和 PC 端的 User-Agent 是不一样的。这个时候我们就可以在 Vary 里声明判断 User-Agent一样的时候才使用同一个缓存，如果不一样则分别使用两个缓存。

Vary 头是很重要的，如果我们使用代里缓存，很多情况下都会使用到 Vary。因为不是只根据 url 进行缓存就够了，很多时候会有不同数据返回的场景。

nginx 代理缓存还有很多功能需要进行提升的，比如：

  • 写脚本让缓存使用内存数据库
    这样它的搜索性能会更高，因为 nginx 默认缓存是写在磁盘上的，读写磁盘会降低效率

4-3 HTTPS 解析

加密

  • 公钥
  • 私钥

  1. 客户端先生成一个随机数传输到服务端，中间会带上客户端支持的加密套件（有很多不同的加密方法可选）；
  2. 服务端拿到随机数之后先存着，服务端也生成一个随机数，这个随机数会伴随着服务端的证书（也就是服务端的公钥）一起传输给客户端；
  3. 客户端拿到随机数之后也先存着，通过服务端传给它的公钥生成一个预主秘钥，然后将预主秘钥使用服务端的公钥加密传输给服务端（这个过程就是一个没有办法被中间人解析的过程，因为它使用的是服务端的公钥加密的，只有服务端有私钥可以解密）；
  4. 加密的预主秘钥传输到服务器之后，服务器使用私钥解密，得到预主秘钥。
  5. 客户端和服务端分别对客户端随机数、服务端随机数、预主秘钥进行算法操作（之前约定好的加密套件里的一种加密方式）生成一个主秘钥。
  6. 后期传输的数据全部通过主秘钥加密，保证数据的安全性。
