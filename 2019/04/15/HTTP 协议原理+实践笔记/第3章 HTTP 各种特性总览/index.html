普通跨域请求必须在被跨域方设置跨域头

3-4 缓存头Cache-Control的含义和使用

可缓存性

  • public：http 经过的任何地方都可以执行缓存
  • private：只有发起请求的浏览器才能执行缓存
  • no-cache：任何节点都不可以执行缓存，每次进行缓存前都需要去服务器验证一下，如果服务器返回的结果告诉你，你可以执行缓存，才能执行

到期

  • max-age=
  • s-maxage=：只在代理服务器中生效
  • max-stale=

重新验证

  • must-revalidate
  • proxy-realidate

其他

  • no-store：本地和代理服务器都不可以使用缓存
  • no-transform：告诉代理服务器不要随便改动返回的内容

这些头只是一个声明性的作用，没有任何强制的约束力

Etag

3-7 HTTP长连接

HTTP 请求是在 TCP 连接上发送的，一个 TCP 连接可以发送多个 HTTP 请求，但是在 HTTP1.1 里面，HTTP 求情在 TCP 连接里发送是有先后顺序的，所以为了提高性能还是得使用并发 TCP 连接的方式。但是在 HTTP2里面，因为可以在一个 TCP 连接里面并发 HTTP 请求，所以只开一个 TCP连接就够了。

3-8 数据协商

分类

  • 请求

请求

  1. Accept
    • Accept：指定数据类型
    • Accept-Encoding：指定数据编码格式，主要是为了限制服务端选择数据压缩的方式
    • Accept-Language：指定返回内容的语言
    • User-Agent：表示浏览器的相关信息
  2. Content
    • Content-Type：实际返回的数据格式
    • Content-Encoding
    • Content-Language
