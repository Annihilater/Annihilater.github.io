第9章 高级部分：面向对象

python
入门与进阶
发布日期:   2019-04-15
更新日期:   2019-12-06

本章讲详细讲解面向对象的概念。包括面向对象的三大特性（继承、封装、多态）、类的基本构成元素、python的内置类属性、方法重写、运算符重载、静态方法等

9-1 类的定义

  • 类()里传入的参数和函数()里传入的参数是不一样的；
  • 类最基本的作用就是封装代码；
  • 类下面的函数一定要传入self，否则在调用的时候会报错；
  • 在函数的内部使用类定义的变量，必须使用self.variable，否则也会报错；
  • 类只负责定义对象及其行为，不负责执行，需要在类的外部执行类的行为，在类的内部调用类的行为（方法或者函数）是错误的。

9-2 浅谈函数与方法的区别

其实方法和函数并没有明确的界限。区别只是在理解上的区别。

方法：是面向对象里的概念，面向对象更加关注的是设计，所以方法是属于设计层面的。

函数：是面向过程里面概念，程序运行、过程式的一种称谓。

变量：当变量直接出现在模块下面，我们称之为变量；

数据成员：当变量出现在类下面，在 python 里更多的称之为数据成员。

数据成员这样的一个词汇更多体现的是类的封装性。

9-3 类与对象

类：是现实世界或思维世界中的实体在计算机中的反映，它将数据以及这些数据上的操作封装在一起。

行为没有找对主题是很多同学在设计面向对象的时候，最容易忽略的问题，这也是导致你的面向对象的设计不合格的主要原因之一。

类就像是模板一样，通过类可以生产很多实例对象。

9-4 构造函数

作用：让类模板生成不同的对象。

构造函数只能返回 None，不能强制返回其他类型，否则会报错。

强制给__init__返回非None类型会报错

9-5 区别模块变量与类中的变量

模块变量：是写在模块下面的，模块变量的作用域是整个模块

类变量：是写在泪下面的，类变量的作用域是类

9-6 类变量与实例变量

类变量：是和类相关的变量

实例变量：是和对象相关的变量

9-7 类与对象的变量查找顺序

上例尝试打印实例对象 student1 不存在的变量 name，但是打印出来的是类 Student的变量 name。

原因：当我们尝试去访问实例变量，Python首先会在对象的实例变量列表里去查找到底有没有 name变量，如果没有的话Python并不会返回 None，Python会继续到类变量列表里寻找。如果在类变量里也没用找到，Python会到该类的父类里继续寻找，一层一层的往上查找，直至找完，最后还是没有找到的话会报错AttributeError: 'Student' object has no attribute xxx。

9-8 self与实例方法

  • 如果在类下面创建一个实例方法的话，必须在方法的第一个参数传入 self。这个 self是 Python规定的参数，和我们自定义的参数是不同的。这个 self的名字可以随意更改，Python会默认第一个参数为 self
  • self代表的实例对象本身，跟类是没关系的

构造函数可以看做特殊的实例方法。构造函数和实例方法的区别：

  1. 调用的方式不同
  1. 意义不同
    • 实例方法的主要作用是用于描述类的行为
    • 构造函数的意义在于初始化类的各种特征

上节的示例使用 self就可以正确的给实例变量赋值

9-10 类方法

类方法：在定义方法的 def上面加上装饰器@classmethod

  • 类方法传入的第一个参数是 cls，很容易看出 cls就是 class的简写，cls代表的是类本身
    这个 cls也可以改成其他任意的名称，Python会默认第一个参数为 cls
  • 主要作用：操作与类相关的变量

实例方法与类方法的区别：

  • 实例方法关联的是实例对象 self
  • 类方法关联的是类 cls，类方法前面有@classmethod

误区：

很多同学会认为，既然我们可以在实例方法中操作类变量，还要类方法干什么？

这里需要注意的是，这不是一个语法或者功能上的满足的问题，而是你要操作一个和类有关的变量，最正确的方式还是使用类方法。并不是说你使用实例方法不对，但是很多时候在意义上是说不通的。

9-11 静态方法

静态方法：需要在定义方法 def上面加上装饰器@staticmethod

  • 静态方法不强制传入参数（类方法需要传 cls，实例方法需要传 self）
  • 类、实例对象都可以访问静态方法
  • 静态方法内部可以访问类变量，但是不能访问实例变量

七月心语：

不建议经常使用静态方法，因为静态方法语面向对象的关联性非常弱，跟类和对象关系不大，它和普通的函数几乎是没有任何区别的。

9-12 成员可见性：公开和私有

类的变量是类非常重要的特征数据，所以说如果要修改一个类的特征数据，不应该通过直接访问类变量的方式改变变量的状态，而是应该通过类自己的方法来完成对类变量状态的更改。

原因：我们可以在方法里对传入的参数做一些判断，如果传入的参数不符合数据状态的要求，就会拦截并返回适当的信息，这样就对数据起到了保护作用。

七月心语：

规范确实是这样的，但是实际情况可能会有所不同。如果一个类下面有几十个变量的话，我们要给每一个变量配一个类似与 marking这样的方法来做变量读写的话，其实是一件非常麻烦的事情，有时候我自己写代码的时候，为了偷懒，我也不会去写类似 marking这样的方法，有时候就直接访问类变量修改类变量的状态了，那是因为我非常类的内部结构才会做出这样的操作。

那么这个 marking方法的意义是什么呢？

这个作用就在于，如果你的类是提供给别人使用的，别人是不知道你的类内部机制和规则的。他也许会认为你的 score可以是负分，所以这种情况下，通过类的方法来操作类变量的方式是很有必要的。

私有：

一般地，如果变量或者方法的开头是双下划线__，Python都会认为这个变量或者方法是私有的。

如果一个方法或者变量开头和结尾都有双下滑线__，Python 则不会认为该变量或者该方法是私有的。

尽量不要将变量名前后都加上双下划线，因为这是 Python内置变量的命名方式。

9-14 继承

Python允许多继承：一个子类可以继承多个父类。

代码解释：Human.__init__(self, name, age)

Human是一个类，Human调用了__init__构造函数，构造函数的实质是一个实例方法，一个类去调用一个实例方法从面向对象的理解方面来讲是不正确的，类是抽象的，对象是具体的，你用一个类去调用一个实例方法是说不通的。但是 Python的灵活就灵活在这里，它虽然认为你的这种做法是没有意义的，但是他并不限制你强行这么做，所以说就出现了用类来强行调用实例方法的特殊的情况。

为什么在这里需要显式的传入 self，而在实例化的时候不需要传入 self？

区别在于实例化的时候确实调用了构造函数，这个调用构造函数的调用方是谁？它不是我们自己去调用构造函数，是 Python内部实例化的机制，自动帮我们调用构造函数。所以说在这样的机制下面 Python会帮助我们传入 self。但是再回到当前情况，Human.__init__(self, name, age)就是用类调用了一个实例方法，和实例化的时候调用构造函数是决然不同的，这种情况下面就是一个普通的方法调用，普通方法的调用当然需要把所有的参数都传入进去，不能遗漏 self，没有人会帮你传入 self。

那么为什么我在使用对象调用do_homework的时候不需要传入 self呢？

原因在于调用do_homework的时候并不是通过 Student这个类去调用的，我们是通 student1这个实例对象去调用的，self的实质指代的就是实例对象，所以通 student1这个实例对象去调用的时候，Python是知道这个 self指的是谁的，指的就是 student1这个实例对象，Python会自动的把 student1传入到do_homework里面来。
